Database Topics

数据库专题

从表设计、SQL、索引和事务开始，逐步走向缓存、分库分表和数据一致性，不再只会背名词。

Core

必须打牢的四块

表设计

  • 主键、外键、范式与反范式。
  • 一对多、多对多与中间表。
  • 时间字段、状态字段和审计字段设计。

SQL

  • 增删改查、联表、聚合、分页。
  • 子查询、视图、存储过程理解即可。
  • 能把业务需求翻译成查询语句。

索引

  • B+ 树、联合索引、最左前缀。
  • 回表、覆盖索引与索引失效。
  • 慢查询排查与执行计划基础。

事务与锁

  • ACID、隔离级别、脏读与幻读。
  • 行锁、表锁、乐观锁与悲观锁。
  • 死锁与并发更新问题。

Advanced

进阶应用技术

Redis

理解字符串、哈希、列表、集合、有序集合，以及缓存击穿和过期策略。

读写分离

理解主从复制、延迟、读一致性和典型适用场景。

分库分表

理解数据量增长后的拆分动机、路由方式和复杂性成本。

一致性

理解缓存与数据库双写、最终一致性和补偿思路。

Interview

数据库高频面试点

为什么索引能加速查询？

要讲数据结构、扫描范围缩小和回表成本，而不是只说“因为有目录”。

联合索引为什么要看最左前缀？

要能结合索引排序结构和查询条件使用方式来解释。

事务隔离级别怎么选？

要讲一致性要求、并发性能和业务风险，而不是只背四个级别名字。

Redis 和 MySQL 各自适合什么？

要从持久化、查询能力、延迟和业务场景来回答。
