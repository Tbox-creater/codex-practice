Systems

把网络基础和常见应用技术连起来学。

这一页聚焦最常用的系统知识，不追求面面俱到，只覆盖开发中最容易遇到、也最值得反复复习的主题。

优先顺序

  • 先学 HTTP 和请求链路。
  • 再补 TCP/IP、DNS、HTTPS。
  • 最后进入缓存、消息队列、监控这些应用层能力。

HTTP / HTTPS

  • 请求方法、状态码、Header、Cookie、Session。
  • HTTPS 做了什么，证书和加密解决了什么问题。
  • 开发里最常见的跨域、鉴权和缓存头问题。

TCP/IP / DNS

  • 知道网络分层，不必一开始死磕全部细节。
  • 理解域名解析、连接建立、数据传输的大致过程。
  • 能够把“接口慢”“连不上”“超时”这类问题和网络链路联系起来。

缓存与消息队列

  • 缓存适合解决读多写少、热点数据和响应速度问题。
  • 消息队列适合削峰、解耦和异步处理。
  • 学习时先理解场景，再记技术名词。

监控与日志

  • 会看应用日志、访问日志和错误栈。
  • 知道监控最少要看可用性、耗时和错误率。
  • 线上排错时先定位范围，再下结论。
