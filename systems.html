Networks And Applied Tech

网络与应用技术

把网络层知识和常见业务技术拆开学，避免只会背协议名词，却不知道在实际项目里什么时候用。

Network

网络层学习主线

HTTP / HTTPS

  • 请求方法、状态码、幂等性、缓存控制。
  • Cookie、Session、Token 与跨域。
  • HTTPS 握手、证书、常见安全风险。

TCP / IP

  • 三次握手、四次挥手、流量控制、拥塞控制。
  • 为什么会粘包、丢包、重传、超时。
  • DNS 解析与从域名到页面的完整链路。

实时通信

  • 轮询、长轮询、SSE、WebSocket 的区别。
  • 什么时候选消息队列，什么时候直接推送。
  • 聊天、通知、实时看板的常见实现思路。

网关与鉴权

  • Nginx、反向代理、负载均衡和限流。
  • JWT、OAuth、单点登录的适用场景。
  • 接口签名、重放攻击和权限边界。

Applied Tech

应用技术路线

缓存

理解浏览器缓存、CDN、Redis、本地缓存，以及缓存穿透、击穿、雪崩。

消息队列

理解异步削峰、解耦、重试、顺序消费、幂等处理和死信队列。

搜索

理解倒排索引、全文检索、分词、排序和推荐与搜索的边界。

日志与监控

理解访问日志、应用日志、指标、链路追踪和告警，知道故障时先看什么。

Questions

你应该能回答的知识点

为什么浏览器输入 URL 后会看到页面？

至少要能讲清 DNS、TCP 连接、TLS、HTTP 请求、服务端处理、浏览器解析渲染这几段。

为什么需要缓存，缓存会带来什么问题？

要能讲性能收益，也要能讲一致性、失效策略和热点问题。

为什么消息队列能削峰？

要能把“请求高峰”和“消费速度”分开理解，而不是只会背定义。

JWT 和 Session 的取舍是什么？

要能从服务端状态、扩展性、安全性和主动失效来回答。
